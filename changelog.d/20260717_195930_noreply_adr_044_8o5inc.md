### Added

- **`PolicyFile.internal_namespaces`** — project-configurable internal-
  implementation-namespace convention for suppression reachability tagging
  (ADR-044 P1 item 5). Projects using a convention other than the default
  `detail`/`impl`/`internal`/`__detail`/`_impl` five-token set can now set
  `internal_namespaces: [priv]` in a policy file; it threads through
  `MarkReachability`/`DetectInternalLeaks`/`DemoteUnreachableInternalChurn`.

### Fixed

- **Suppression diagnostics now reach two remaining `checker.py` call
  sites** (ADR-044 P1 item 6) — SONAME/platform-floor policy findings
  (`_filter_suppressed_changes`) and ADR-027 surface-metric roll-ups
  (`_apply_surface_metrics`) now route through `SuppressionList.evaluate()`
  instead of the boolean `is_suppressed()`, so a broad rule matched but
  withheld by the reachability/`allow_public_break` gate emits the same
  `suppression_would_hide_public_break` diagnostic on these paths that
  `ApplySuppression` already produces for changes it sees directly.
