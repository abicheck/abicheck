### Fixed

- **C++20 detection now recursively evaluates nested `#if` chains inside
  a reachable arm, instead of blindly copying them through.** The
  preprocessor-guard masker used a flat depth counter for any `#if`
  nested inside a live arm — it tracked depth well enough to find the
  matching `#endif`, but never evaluated the nested condition itself. A
  dead nested `#if 0` compatibility stub inside an otherwise-live arm
  could therefore still shadow a genuine, active C++20 declaration
  elsewhere in the header (and, symmetrically, live-looking nested code
  inside a dead ancestor arm was already handled correctly). Rewritten
  as an explicit per-depth stack so every nesting level is masked with
  the same rules as the top level, recursively.
