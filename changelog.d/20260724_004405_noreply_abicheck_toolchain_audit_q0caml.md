### Fixed

- **C++20 detection now recognizes a lambda parameter constrained by a
  project-defined concept (`[](MyConcept auto x) {}`) as a genuine C++20
  signal.** It was previously excluded the same way a *bare* `auto`
  lambda parameter is (valid since C++14) — but a *constrained* lambda
  parameter is exactly as C++20-only as the equivalent ordinary-function
  form, so a header whose only use of the feature was a constrained
  generic lambda was silently parsed without `-std=gnu++20`.
