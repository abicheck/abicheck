### Added

- **Call-graph reachability evidence** (ADR-044 P1 items 1-2) ‚Äî suppression
  reachability tagging now also consults the optional L5 source graph's
  `DECL_CALLS_DECL`/`DECL_REFERENCES_DECL` edges (`--sources`/`--build-info`/
  `--header-graph`), not just layout/type-graph reachability. A public
  inline function calling into a removed internal template specialization
  ‚Äî previously invisible to a broad suppression rule, with no evidence at
  all for the tool to reason about ‚Äî is now tagged `public_reachable`, and,
  when the underlying change is already an artifact-proven break, gets a
  new `internal_symbol_required_by_public_api` overlay finding explaining
  the call path.
- **Structured reachability fields in JSON and SARIF** (ADR-044 P1 item 4)
  ‚Äî `public_reachable`, `reachability_kind`, and `reachability_proof_path`
  now appear as first-class fields in both JSON report modes and SARIF
  `properties`, not only inside the `suppression_would_hide_public_break`
  diagnostic's description text.

### Changed

- **`reachability_kind` gained a third value**, `symbol_availability`
  (ADR-044 P1 item 3), for the new call-graph-only reachability signal ‚Äî
  distinct from the existing layout-based `value_embedding`/
  `pointer_or_signature` values.
