### Fixed

- **C++20 auto-detection no longer mistakes a control-flow statement's
  bare `auto` for an abbreviated function-template parameter.** A bare
  `auto` immediately inside a `(` is the same textual shape whether it's
  a genuine parameter (`void f(auto x);`, C++20) or a range-based for
  loop's declaration (`for (auto x : xs)`, valid since C++11), or an
  `if`/`while`/`switch` condition or init-statement declaring a variable
  (`if (auto x = f())`, valid since C++17). Headers using any of these
  pre-C++20 forms were wrongly force-parsed with `-std=gnu++20`. A
  *constrained* placeholder in the same position (`for (MyConcept auto x
  : xs)`) is still detected, since C++20 genuinely permits one there.
