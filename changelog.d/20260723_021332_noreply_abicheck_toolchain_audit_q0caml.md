### Fixed

- **C++20 type-shadow detection now spans the whole scanned header set,
  not just one file at a time.** `_find_cpp20_requirements` receives the
  entire aggregate header set castxml/clang parse together as a single
  translation unit, but the `concept`/`requires`/`consteval`/`constinit`
  shadowed-type-name checks were previously computed per file. A
  compatibility type shadowing one of those keywords declared in a shared
  header (`compat.hpp`) no longer needs to live in the *same* file as the
  ambiguous bare use (`api.hpp`) to be recognized — the scan is now a
  two-pass walk: shadow flags are OR'd across every file first, then the
  per-line requirements scan runs against that aggregate-wide result.
