### Fixed

- **The GitHub Action's `compare` mode now fails loud instead of warning
  and continuing when an `ast-frontend`/`gcc-path`/`gcc-prefix`/
  `gcc-options`/`sysroot`/`nostdinc` input is configured against a
  directory/package (release/bundle) operand.** The per-library fan-out
  never threads that L2 compile context to each pair's header dump, so
  continuing silently could produce a green verdict from headers parsed
  under the wrong macros/sysroot/frontend — the same silent-wrong-result
  risk the evidence-flags guard (`--depth build`/`source`) was already
  fixed to fail loud on. Compare the libraries individually to use these
  inputs.
