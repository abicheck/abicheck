### Fixed

- **The GitHub Action's `validate-inputs.sh` now rejects a
  directory/package `compare` operand combined with
  `ast-frontend`/`gcc-path`/`gcc-prefix`/`gcc-options`/`sysroot`/`nostdinc`
  before any dependency install, mirroring `run.sh`'s fail-loud guard.**
  Previously only `run.sh` caught this, so a workflow with a slow
  dependency-install step still passed the fail-fast validation step and
  only errored after setup began. `action.yml`'s input descriptions and
  the generated `docs/reference/github-action-inputs.md` now also note
  that these six inputs apply to `compare` only for a single-library
  (non-directory, non-package) operand.
