### Fixed

- **The C-vs-C++ auto-detection no longer forces C++ mode purely because
  of C++20 syntax behind a `#ifndef __cplusplus`/`#if
  !defined(__cplusplus)` guard's `#else` arm.** In C mode `__cplusplus` is
  undefined, so that guard's own arm (typically a portable C fallback,
  e.g. `int consteval;`) is the one actually reachable there, not its
  `#else` — the opposite polarity from `#ifdef __cplusplus`. Previously
  the guarded arm was unconditionally masked (correct only for the
  already-decided C++ dialect question), leaving the `#else` visible and
  wrongly forcing C++ mode purely because it contained C++20 syntax.
