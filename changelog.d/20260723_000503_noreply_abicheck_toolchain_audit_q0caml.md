### Fixed

- **Windows CI now installs an actively-maintained CastXML build.** The
  `integration-tests (windows-latest)` job installed CastXML via
  `pip install castxml`, a stale, unmaintained 0.4.5 legacy wheel that the
  `castxml_policy` version gate correctly rejects. It now installs from
  conda-forge (which currently ships CastXML 0.7.0 for `win-64`) using the
  Miniconda already preinstalled on GitHub-hosted Windows runners, matching
  the conda-forge suggestion already documented in
  `action/install-castxml.sh` for non-Linux platforms.
