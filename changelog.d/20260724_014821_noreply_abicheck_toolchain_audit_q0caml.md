### Fixed

- **C++20 detection now recognizes a constrained placeholder *return
  type* using a project-defined concept (`MyConcept auto f();`), not
  just the equivalent parameter form.** The check only ever looked for
  the identifier immediately after `(`/`,` (a parameter position); a
  return type at the very start of a declaration has neither, so the
  only C++20 signal in a header using solely this form went undetected
  and the header was parsed without `-std=gnu++20`.
