### Fixed

- **Fixed header-scoped Mach-O dumps never matching any C++ export** —
  `_dump_macho` applied its own leading-underscore strip to export names
  that `macho_metadata.py` had already normalized, corrupting every
  Itanium-mangled (`_Z...`) symbol by eating its own mangling prefix. This
  meant `dump --header` on a Mach-O `.dylib` with C++ exports could never
  match a declaration to an export, silently falling back to export-table-only
  mode (losing all struct/field/base type information) on every run.
