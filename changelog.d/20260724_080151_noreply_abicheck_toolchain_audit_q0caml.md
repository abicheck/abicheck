### Fixed

- **C++20 auto-detection's transitive quoted-`#include` following now
  respects preprocessor reachability.** A quoted `#include` confined to
  an inactive or C++-only guard (e.g. `#ifdef __cplusplus` in an
  otherwise-C header) was previously followed unconditionally by a raw
  text scan, so the included file's C++20 syntax could wrongly force
  C++20 mode even when the guard around the `#include` itself would
  never be reached. The include is now only followed if it survives the
  same preprocessor-guard masking used for the calling decision (the
  C++-dialect decision vs. the from-scratch C-vs-C++ language-mode
  decision).
