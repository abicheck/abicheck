### Fixed

- **C++20 detection now recognizes a project-defined concept's
  constrained placeholder written with trailing-return-type syntax**
  (`auto f() -> MyConcept auto;`) — valid C++20, since `MyConcept auto`
  is a placeholder-type-specifier usable as a trailing-return-type. The
  text before the concept name ends in the `->` from the trailing-return-
  type arrow rather than a parameter-list boundary or a statement
  boundary, so a header whose only C++20 signal used this form went
  undetected and was parsed without `-std=gnu++20`.
