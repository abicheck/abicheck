### Fixed

- **C++20 auto-detection's transitive quoted-`#include` following now
  handles backslash-continued directives and ignores includes trapped
  inside raw string literals.** A directive split across a
  backslash-newline continuation (`#include` with a trailing backslash,
  filename on the next physical line — valid C/C++, spliced away before
  the preprocessor ever tokenizes it) previously wasn't recognized at
  all, since the pattern only matched within one physical line —
  continuations are now spliced before matching. Conversely, a
  `#include "..."`-looking line trapped inside a raw string literal's
  body was previously followed as if it were a real directive; raw
  string bodies are now blanked before matching (ordinary string
  literals are deliberately left alone, so a genuine include's own
  argument still survives).
